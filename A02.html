Part 1: Directions on Using Git

  1. Download Git: Go to "https://git-scm.com/downloads" and download the appropriate version for your operating system. Follow the installation instructions.
  2. Configure Git: Launch a terminal or command prompt and execute the subsequent commands to set up your Git username and email address:
    
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
  3. To establish a new repository, navigate to your desired directory in the terminal or command prompt. Utilize the following command to initialize a fresh Git repository:
    
git init
  4. Clone an Existing Repository: Navigate to the directory where you want to clone the repository. Run the following command, replacing `` with the URL of the repository you want to clone:
    
git clone
  5. Add Files to the Staging Area: After making changes to your files, you can add them to the staging area using the following command:
    
git add
    Replace `.` with specific file names if you only want to stage certain files.
  6. Commit Changes: Once your changes are staged, you can commit them to the repository with a descriptive message using the following command:
    
git commit -m "Your commit message here"
  7. Push Changes to GitHub: If you're working with a remote repository on GitHub, you can push your committed changes using the following command:
    
git push origin master
    Replace `origin` with the name of your remote repository and `master` with the name of your branch if they are different.

Part 1: Directions on Using WebStorm

  1. Download and Install WebStorm: Navigate to "https://www.jetbrains.com/webstorm/download" and download the appropriate version for your operating system. Follow the installation instructions.
  2. Open WebStorm: Launch WebStorm from your applications folder or desktop shortcut.
  3. Configure Git in WebStorm: Go to File > Settings > Version Control > Git. Enter the path to your Git executable if it's not automatically detected.
  4. Clone a Repository: In WebStorm, go to VCS > Git > Clone. Enter the URL of the repository you want to clone and specify the directory where you want to save it.
  5. Make Changes: Open the cloned project in WebStorm, make changes to files as needed.
  6. Commit Changes: Go to VCS > Commit. Add a commit message describing the changes you've made, then click Commit.
  7. Push Changes to GitHub: After committing changes, go to VCS > Git > Push. Verify the branch and destination, then click Push.

Part 2: Glossary

  • Branch: A parallel version of a repository.
  • Clone: Create a local copy of a repository.
  • Commit: Record changes to the repository.
  • Fetch: Download objects and refs from another repository.
  • Git: A distributed version control system.
  • GitHub: A platform for hosting and collaborating on Git repositories.
  • Merge: Combine changes from different branches.
  • Merge Conflict: A condition that occurs when Git is unable to automatically resolve conflicting changes.
  • Push: Send local commits to a remote repository.
  • Pull: Fetch changes from a remote repository and integrate them into the local repository.
  • Remote: A version of a repository that is hosted on a server.
  • Repository: A location where all the files for a particular project are stored.
